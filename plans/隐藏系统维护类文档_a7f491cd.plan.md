## 三、改动点

### 1. 后端：[前端开发/kb_backend.py](前端开发/kb_backend.py)

**KnowledgeDocument 数据类（约 354 行附近）加字段**

```python
audience: str  # "marketing"(默认) | "ops"
```

**`_load_documents` 解析 frontmatter（约 542 行附近）补一行**

```python
audience=_normalize_text(metadata.get("audience")) or "marketing",
```

**`list_documents` / `search` / `get_overview` / `list_governance_items` 增加 audience 过滤参数**

默认值 `audience="marketing"`，传 `audience="ops"` 才返回 ops 文档，传 `audience="all"` 返回全部（用于诊断）。

需要改的位置（按 grep 出的行号）：

- `list_documents` (约 713 行)：签名加 `audience: str = "marketing"`，遍历时按字段过滤
- `search` (约 926 行)：同上
- `get_overview` / `get_facets`（约 828, 845, 871, 913 行）：统计也要按 audience 分桶

**KnowledgeDocument.to_dict 输出 audience 字段**（约 396 行附近），前端能拿到。

### 2. 后端 API：[前端开发/server.py](前端开发/server.py)

`/api/overview`、`/api/documents`、`/api/search` 三个接口读取 query 参数 `audience`，默认 `marketing`，透传给 repository。

```python
audience = self._optional_param(params, "audience") or "marketing"
```

### 3. 前端导航：[前端开发/static/index.html](前端开发/static/index.html) 与 [前端开发/static/app.js](前端开发/static/app.js)

**index.html 主导航加一个低调入口**（约 38-42 行）：

```html
<a href="#ops" data-nav="ops" class="nav-secondary">系统维护</a>
```

通过 CSS 把 `.nav-secondary` 字号调小、颜色降饱和，放在最右侧。

**app.js 路由处理**（约 137 行 `parseRoute` 附近）：

- 新增 view = `ops` 时，调接口都带 `audience=ops`
- 默认 view（home/browse/search/doc）请求一律带 `audience=marketing`
- URL 上手动带 `?audience=ops` 也支持，便于排查

### 4. 不需要改的东西

- 不动 frontmatter 里的 `canonical_path`
- 不动文件物理位置
- 不动评论存储 `data/comments.json`（已存评论用 path 关联，路径没动）
- 不动审计脚本和命名治理字典
