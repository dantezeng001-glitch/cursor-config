**OpenRun Pro 2 月银白：从一次换色，跑出 PMKT 自己的 SOP 与 Skill**

**S · 背景**
入职第三周接手 OpenRun Pro 2 月银白上市——这是我在韶音第一次主操盘的换色项目。第一周做 Color Name 和 Slogan 时发现一件事：这件事特别"颅内"——输入复杂（产品调性 + 客群洞察 + 渠道偏好 + 历史 slogan 资产 + 品牌口径），过程不可视，最后输出却只是几句话。更要命的是，**同样的判断每个换色项目都要重做一次**，没有沉淀。

**T · 任务**
我给自己设了两个并行目标：① 把这次新色上市本身**完整跑通**，按节点交付 Color Name / Slogan / Storytelling / 素材方向；② **同步把流程用 AI 固化下来**——这是我第一次尝试"边做边固化"，而不是事后再总结。

**A · 行动**

1. **画 As-Is**：在《新手上路月度工作思考——换色上市与 AI》里，画出"人脑 Process"和"AI Process"两条对照流程，把每一个断点（Name → Slogan → Storytelling → 物料规划）和每一个监督点都标出来；
2. **写 SOP**：拆出 Step 0 ColorName / Step 1 Slogan / Step 2 Storytelling / Step 3 素材方向 四份 SOP，每份都明确"断点 = 等用户确认才能继续"；
3. **抽方法论**：再向上抽象一层，写出《流程拆解与 AI 化_方法论 SOP》——不只为换色服务，是任何 PMKT 流程要 AI 化时都能用的元方法；
4. **打包成 Skill**：放进 `pmkt-product-marketing/workflows/color-launch/`，里面是四个 Step，可以被 Cursor、Claude 这类 AI 工具直接调用。

- **金句换为「从一次换色，确认了一条 AI 提效的路径」**——见上方说明
- **A/B 线回扣同步换**：
  - A 线旧版"学到 PMKT 业务的'不可全自动'边界" → 新版"第一次完整经历'消费品换色上市'的母本生产"（A 线本来就该是"学业务"，旧版偏了，借这次校正回来）
