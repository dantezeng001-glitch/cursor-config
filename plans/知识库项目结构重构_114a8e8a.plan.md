### 第二步：修复 4 个 Python 脚本

所有脚本都假设自己在项目根目录运行，且引用旧路径。需要统一改为 **基于脚本自身位置的相对路径**，使用 `Path(__file__).resolve().parent.parent` 定位项目根。

#### [脚本/generate_excel.py](统一表述知识库/generate_excel.py)

- 修改 `BASE_DIR`：从 `Path(__file__).parent` 改为 `Path(__file__).resolve().parent.parent`（因为脚本现在在 `脚本/` 子目录）
- 修改 `DATA_FILE`：`BASE_DIR / "数据" / "knowledge_base.json"`
- 其他引用（`OUTPUT_FILE`, `HISTORY_DIR`）相应调整

#### [脚本/check_consistency.py](统一表述知识库/check_consistency.py) — 需要重写数据加载逻辑

- **问题**：仍在 `import generate_excel.py` 读取 Python 数组（`gen.TERM_GLOSSARY` 等），但数组已不存在
- **修复**：改为从 `数据/knowledge_base.json` 加载 JSON，将 `check_list()` 函数改为接收 dict list
- 修改 `SEARCH_DIRS`：`源文件/Meet&Spec`、`源文件/POP`、`新文件收件箱`

#### [脚本/fix_no_source.py](统一表述知识库/fix_no_source.py) — 需要大幅重写

- **问题**：仍在 import generate_excel.py 读取数组，然后用正则回写 generate_excel.py 的 Python 源码
- **修复**：改为从 `数据/knowledge_base.json` 读取 JSON，清理后直接写回 JSON 文件
- 这大幅简化了脚本（不再需要 `replace_array()` 等正则替换逻辑）
- 修改 `MEET_DIR` / `POP_DIR`：指向 `源文件/Meet&Spec`、`源文件/POP`

#### [脚本/auto_extract.py](统一表述知识库/auto_extract.py) — 需要重写数据加载逻辑

- **问题**：用正则从 generate_excel.py 的 Python 代码中提取已知词条
- **修复**：改为从 `数据/knowledge_base.json` 读取 JSON，直接遍历 dict 获取 EN 词条
- 修改 `MEET_DIR` / `POP_DIR` / `INBOX_DIR`：指向新路径
