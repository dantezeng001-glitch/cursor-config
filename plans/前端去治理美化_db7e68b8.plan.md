# 前端去治理与美化计划

## 改造判断

按你的选择，治理信息只保留在系统内部，不再出现在网站前台。实现上优先改展示层，避免动知识库文件；后端只处理前端不再需要的请求链路，减少加载时的治理概览请求。

## 主要改动

- 在 `[前端开发/static/index.html](前端开发/static/index.html)` 删除主导航「治理看板」、侧栏「治理快照」和使用说明里的治理说明；副标题改成浏览、搜索、来源追溯、评论。
- 在 `[前端开发/static/app.js](前端开发/static/app.js)` 移除前端治理入口：不再加载 `/api/governance/overview`，`#governance` 旧路由跳回首页或浏览页；首页删「治理健康度」「治理待确认」「自动迁移候选」。
- 在 `[前端开发/static/app.js](前端开发/static/app.js)` 清理详情页和列表里的治理字段：去掉「进治理看板」「治理状态」「治理动作」「自动迁移资格」「置信度」「治理提示」「推荐动作」「待确认原因」等前台文案；保留文档 ID、当前路径、负责人、更新时间、相关产品、相关概念、来源追溯和评论。
- 在 `[前端开发/static/app.css](前端开发/static/app.css)` 做视觉重排：更轻的背景、更明确的首页 hero、更精致的卡片阴影/边框、详情页头部主次层级、元数据卡片降噪、移动端按钮和列表更顺。
- 后端只做效率相关检查：确认前端不再调用治理 API；如无必要，不删除 `[前端开发/server.py](前端开发/server.py)` 和 `[前端开发/kb_backend.py](前端开发/kb_backend.py)` 的治理接口，避免扩大回归面。若发现前端仍因 summary/detail 返回治理字段造成明显冗余，再收窄 JSON 输出并同步测试。

## 验证方式

- 运行现有后端测试：`python -m unittest discover 前端开发/tests`。
- 启动本地服务后手动检查首页、浏览、搜索、文档详情：前台不出现「治理」「迁移」「置信度」「待确认队列」等治理词；搜索、打开文档、来源追溯、评论仍可用。
- 检查移动端宽度下导航、搜索框、详情页按钮不挤压。
