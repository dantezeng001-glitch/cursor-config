# PMKT × 用研 桥接机制

## 核心判断

断裂的根因不是某一侧出了问题，是两边没有**共同的工作语言**：用研的章节结构是"产品力 / 人群 / 功能验证"，PMKT 的交付物结构是"FABE / Message House / Slogan / Copy / POP / Listing / Meet&Spec"，中间缺一个**翻译层**。

方案分三层，主力是翻译层，协作机制和共读会是配套。

## 核心工具设计：Insight→Action 拆解矩阵

### 结构

双轴表格。

**行（用研维度，6 条 MECE）**

- 卖点优先级（推荐/不推荐 Top 排名 + 各维度评分）
- 目标人群（画像、品牌关系、年龄、性别）
- 购买旅程（知晓渠道、念起时刻、吸引卖点、顾虑、购前行为）
- 场景与用法（高频高满意度场景、替代原耳机场景、跨品类场景）
- 顾虑与反驳（分人群的担忧点、典型差评、抱怨高频词）
- 区域差异（北美 vs 英德，或目标市场差异）

**列（PMKT 交付物，9 列，贴合现有 SOP）**

- FABE 卖点矩阵
- Message House 主线 / 分主题
- Slogan 候选池
- 英文 Copy（长/中/短）
- 包装文案 EN
- POP 文案
- Listing 文案 & A+ 页面结构
- Meet&Spec 推介话术
- 投放人群包 / 创意方向

**每个交叉格子固定三字段**

- 动作（改什么 / 加什么 / 砍什么，动词开头）
- 具体建议（可直接落到交付物里的文案级别建议，不能是"加强 XX"这种抽象话）
- 证据引用（来源章节 + 具体数据，比如"§一.推荐Top1 北美 70.3%"）

### 填法示范（摘自 NT 样例）

行 = 卖点优先级 × 列 = FABE：

- 动作：把音质锁为 Top1 卖点，滤噪从 Top1 候选降为 Top3 支撑
- 具体建议：FABE 表第 1 行 Feature = "DualPitch 双单元音质"；滤噪作为 F4，从"全天候主打卖点"改为"场景化辅助卖点（工作学习 / 通话会议）"
- 证据：§一 推荐 Top1 音质（北美 70.3% / 德 65.7% / 英 56.7%）；滤噪仅 Top4-6，占比 16%-27.8%；不推荐里"滤噪效果差"排 Top3

行 = 顾虑与反驳 × 列 = Listing：

- 动作：在 Listing A+ 页面加入"嘈杂环境听不清"的反驳模块
- 具体建议：针对耳挂新用户/开耳非耳挂用户（抱怨比例 7%-13%），加一段"在 XX 场景下 NT 的滤噪能做到 XX dB 降低"+ 场景图
- 证据：§一 不推荐 Top1 北美 / §一 嘈杂环境听不清-三类用户抱怨比例表

## 协作机制：轻量改动用研那端

**不改用研报告结构**，只改两件事：

- **PMKT 项目启动时给用研下单**：用 `02-PMKT对用研的Brief下单模板.md`，包括要回答的营销决策（比如"NT 下一代的 Slogan 主线应主打音质还是滤噪"）、需要的数据颗粒度（"卖点吸引力需拆耳挂老/新/开耳非耳挂三类用户"）、时间要求
- **报告产出时加 2 页 PMKT 摘要**：用研在正常报告外，加一份"对 PMKT 的 2 页摘要"，结构是"3 个最强洞察 + 3 个明确不建议做的事 + 3 个待验证问题"

## 共读会 SOP

用研报告产出后 1 周内，PMKT + 用研必开一次 1-2 小时共读会。议程固定：

- 15 分钟：用研讲 3 个最强洞察（只讲结论和证据，不讲方法）
- 45 分钟：PMKT 主导填拆解矩阵（行逐条过，证据能不能落到列上）
- 15 分钟：对齐优先级（哪些动作本项目必做、哪些下一代留用、哪些待更多数据）
- 收尾：产出一份《XX 报告共读纪要 + 行动清单》，不产出新报告

## 时间钩子：嵌入 PMKT 现有流程

在 [01-PMKT上市核心交付物标准清单.md](PMKT 上市项目梳理/01_核心SOP与指导手册/01-PMKT上市核心交付物标准清单.md) 的时间表里增加 4 个用研 Checkpoint（作为软节点，不新增硬节点，避免流程变重）：

- **T-8月（产品基础信息包阶段）**：用研交付"上一代/同类产品核心洞察摘要"——给 FABE 起草做前置输入
- **T-6.5月（FABE 草稿 V1 前）**：卖点优先级数据复核 Checkpoint
- **T-5月（Message House / Slogan 锁定前）**：目标人群画像 + 核心场景确认 Checkpoint
- **T-2.5月（Meet&Spec / Listing 启动前）**：顾虑与反驳话术输入 Checkpoint

不改硬节点、不改责任人，只是在既有软节点前标记"用研输入 Checkpoint，未到位需启动催单"。

## 落地路径

**不做全流程规范，做最小闭环试点**：

1. 先做 00-05 六份 Markdown 文档（含 NT 样例矩阵，让人一看就知道填法）
2. 选一个在跑的 PMKT 项目（NT 下一代 / XS / NCE，由用户指定）做试点
3. 在试点里跑完一次：下 Brief → 开共读会 → 填矩阵 → 输出动作清单 → 回看 PMKT 交付物是否真的采纳
4. 试点完做一次复盘：矩阵结构是否合用、共读会时长是否合理、Brief 模板是否缺字段
5. 复盘后把流程固化，并考虑是否纳入 [PMKT 上市项目梳理](PMKT 上市项目梳理/) 主索引

## 需要你确认的两个点

- **试点项目**：先选哪个项目做第一次跑通？NT 下一代 / XS / NCE / 其他在跑项目
- **"用研那端"的参与度**：协作机制那层（Brief 下单模板 + 2 页摘要）是否需要在出模板前先和用研团队对一下？还是先 PMKT 侧自己定出来，再拿着草稿找用研对齐

这两点不影响现在动笔写矩阵模板和 SOP，但影响第 5 份《试点手册》的具体内容和第 2 份《Brief 下单模板》的沟通节奏。
