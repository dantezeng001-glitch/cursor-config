## 一、核心思路

把断裂拆成三层错位，对症下药：

- **产出格式错位**：用研给报告，PMKT 要的是能直接填进 FABE / Message House / POP / Meet Sheet / Listing 的字段 → 建 **MIB（Marketing Input Brief，营销输入简报）** 模板，强制用研在发报告的同时附一份 MIB
- **术语体系错位**：NPS/Kano/推荐 Top vs F-A-B-E/主卖点/利益点 → 建 **术语映射表**，一页纸
- **时序错位**：FABE 在 T-6.5~~T-5 月要出，用研只做上市后研究 → 在 PMKT SOP 里加 **用研协作轴**，明确 T-9 月《研究问题单》→ T-8~~T-7 月《前置 MIB V0》→ T-5 月 V1 → T+3 月《上市后 MIB》

## 二、MIB 模板要长什么样（关键）

五张卡片，每张直接对应一个 PMKT 交付物字段，每条结论必须带量化证据和来源。以 NT 报告为例倒推：

**1. 卖点优先级卡（喂 FABE）**

- 主卖点 / 次卖点 / 差异化卖点 三档
- 每条带：量化证据（%/n）、数据来源（国家/样本量）、Kano 分类
- 例：音质 = 主卖点（北美推荐 Top1 70.3%，n=253，期望属性）；滤噪 = 差异化卖点（50%+ 认为是魅力属性，美国用户原声"super excited"）

**2. 目标人群分层卡（喂 Message House + Meet Sheet 受众页）**

- 不用用研常说的"品牌新/老用户"维度，改成 PMKT 能用的"核心购买者 / 被吸引者 / 顾虑者"三档
- 每档带：画像、触达路径、吸引点、顾虑
- 例：北美耳挂老用户（男 30-60 岁，中高收入，办公/外勤商务）→ 滤噪驱动 → 顾虑性价比

**3. 场景-利益点矩阵（喂 Copy / POP / Meet Sheet 场景故事）**

- 横轴场景（工作学习/通话会议/跑步骑行...），纵轴该场景里拉升替代率最强的卖点
- 例：英德工作学习场景滤噪显著拉替代率（UK 骨传导 14.3%→42.9%）→ 办公场景主线

**4. 顾虑与反驳卡（喂 Listing FAQ / Meet Sheet Q&A）**

- Top 不推荐原因 + 每条建议的 PMKT 应对方向（文案预防 / 产品回传 / 受众排除）
- 例：嘈杂环境听不清（不推荐 Top1）集中在耳挂新用户 → 文案明确"适合办公/家居，不主推地铁通勤"

**5. 用户原声库（喂 Copy / VOC / KOL 话术）**

- 每条原声带：人群标签 + 场景标签 + 卖点标签 + 国家
- 直接从报告的"用户原声"表格抽，按标签归档

## 三、时序对齐：用研协作轴挂进 PMKT SOP

挂在现有 [PMKT 上市项目梳理/01_核心SOP与指导手册/02-PMKT上市全链路SOP与协作流程.md](PMKT 上市项目梳理/01_核心SOP与指导手册/02-PMKT上市全链路SOP与协作流程.md) 的阶段一到阶段四里，作为并行的"用研协作"小节：

- **T-9 月（阶段一之前）**：PMKT 交《研究问题单 Research Brief》给用研，写明下一代产品我需要什么结论、什么时候要
- **T-8~T-7 月**：用研基于历史数据 + 桌面研究 + 必要定性访谈交《前置 MIB V0》。注意：**不等新品完整研究做完再给**，数据不全时用置信度分档写结论
- **T-5~T-4 月**：用研补《前置 MIB V1》
- **T+1.5~T+3 月**：用研做上市后完整研究
- **T+3 月**：用研交《上市后 MIB》——用途一喂本代产品的中后段 Listing / 内容优化，用途二喂下一代 PMKT 的《前置 MIB V0》

## 四、两个强制会议

不是每周同步，是卡在关键节点的硬约束：

- **立项后 1 小时《研究问题单共建会》**：双方当场约定哪些结论在哪个时间点给，签一份双方都认的清单
- **每次交 MIB 时 45 分钟《MIB 交接评审会》**：PMKT 按五张卡当场"认领"每个洞察，认领后进 PMKT 任务表；不认领的当场说原因（数据不可信 / 不适用 / 结论冲突），用研带回

## 五、试点策略

**用现在这份 NT 上市后报告回溯生成一份 MIB**，三个目的一次达成：

- 验证 MIB 模板填得出来 / 有没有字段漏洞
- 暴露 NT 报告本身缺什么（比如没有按"核心购买者 vs 顾虑者"分层的维度）
- 给用研团队一个"这就是我要的样子"的范本，比口头解释高效十倍

## 七、推进节奏建议

- 第一步：先出 `01-MIB模板.md` + `02-用研术语映射表.md` + `04-NT回溯样例.md`（这三个是"实物"，能让用研团队看到"具体要我交什么"）
- 第二步：拿这三份和用研团队开一次对齐会，让他们改模板字段（他们会比 PMKT 更知道什么数据是能稳定产出的、什么是产不出来的）
- 第三步：改完后做 `03-用研协作轴SOP.md` + `README.md`，此时时序和 SLA 是两边都认的
- 第四步：下一代产品试点跑一轮，跑完回头合入 PMKT 主 SOP

## 八、风险与应对

- **用研团队抗拒多出一份 MIB**：MIB 不是"多写一份报告"，是"把已有结论按 PMKT 格子重摆一次"，工作量主要是抽取不是写作；NT 回溯样例就是为了证明这一点
- **数据不全时的置信度标注**：MIB 每条结论强制带样本量、国家、时间；数据不全时写"方向性判断，待 X 研究补充"，不是"不写"
- **流程跑起来又退化**：两个强制会议不是"建议开"，是"不开 MIB 不生效"；PMKT 有权拒绝领取没开评审会的 MIB
