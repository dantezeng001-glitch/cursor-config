# Manus 在线 AI 文档校对流程与 Prompt 升级计划

既然明确了直接使用在线 AI 工具（如 Manus），且上游已经能够提供**《变更报告》**，这为我们极大降低了技术开发成本，同时大幅提升了 AI 的可用性。

基于您提供的 `Manus在线文档校对_Prompt设计.md`，目前的 Prompt 逻辑是“拿到全量信息源后去全量找不同”。我们现在需要将这个 Prompt 进行**结构性升级**，将核心逻辑转变为**“基于变更报告的增量/定向排查”**。

### 核心升级方向

1.  **引入《变更报告》作为最高优先级输入**：
    *   在 Prompt 中增加指令：让 AI 第一步必须先解析《变更报告》。
    *   改变比对策略：如果存在变更报告，AI 不需要逐行通读 18 个 Sheet 去对比全量参数，而是**顺藤摸瓜**——如果报告说“重量改为 6.5g”，AI 就只去所有的 Sheet 里搜刮“重量”相关的中英文，检查是否都同步成了 6.5g。这能省去 80% 的无效算力。
2.  **挂载《统一表述知识库》**：
    *   Prompt 需要增加指示，要求 AI 读取 `数据/knowledge_base.json`（或导出的文本版）。
    *   增加一条硬性规则：在校对增量字段的同时，顺手校验该字段的中英文翻译是否符合知识库的标准规范。
3.  **强化“免责”护城河**：
    *   在 Prompt 中明确规定：如果某处变更导致下游出错，但《变更报告》中并未提及此项变更，AI 需要将其特殊标记为“上游隐性变更/未申报变更”。

### 后续行动步骤

1.  **重构 Prompt 文档**：我将修改 `说明书重复校对需求-丹丹/Manus在线文档校对_Prompt设计.md`。
    *   加入对《变更报告》的解析流程（阶段 0 升级）。
    *   将原本的“阶段 2：全量差异检查”拆分为“优先增量检查”+“兜底全量检查”。
2.  **编写极简操作 SOP**：为丹丹等实际执行人输出一个三步走的 SOP：
    *   准备物料（目标表格 + 信息包 + 变更报告 + 知识库）
    *   复制 Prompt 发送给 Manus
    *   基于 Manus 输出的报告在钉钉里做勾选确认。
