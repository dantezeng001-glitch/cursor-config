## 二、框架完善（在你的雏形上做两点扩充）

你的雏形："输入物料 + AI 思考框架与行动指南 + 输出格式与内容 = 人做事的方式"。

我建议扩到 **四格**，加一格在最前面：

| 维度 | 含义 | 对应"人做事" |
|---|---|---|
| **① 任务边界** | Skill 要做什么 / 不做什么 | 接活前先想清楚"我管哪段、不管哪段" |
| **② 输入物料** | 待处理对象 + 参考标尺 | 准备好"今天要处理的东西"和"对照标尺" |
| **③ AI 思考与行动指南** | 执行步骤 SOP + 判断铁律 | 心里那套"按什么顺序、按什么标准做"的方法论 |
| **④ 输出格式与内容** | 交付物长什么样、字段是什么 | 交活前规定"长什么样才算交完" |

**为什么补"任务边界"**：两个案例最珍贵的设计点恰恰在边界——专有名词库"不改原 Excel、不做内容判断"；说明书校对"只读不改、只用 Product Info、只管北美"。没有边界的 Skill 会乱跑，这是读者最容易忽略但最关键的一格。

文档收尾呼应一句话：**Skill 不是新东西，它就是把你脑子里那套"做事的方法论"显性化、固定化，让 AI 能跑、让同事能接。**

## 三、文档结构（5 块）

### 0. 开篇（约 200 字）
不重复定义 Skill（读者看过 karl 提纲）。直接提出文档主张：karl 说 Skill = 提示词 + 可复用资源，本文换一个更容易上手的视角——**Skill 其实就是把"人做一件复杂事的标准动作"写下来交给 AI**。这个视角下，做 Skill 就是回答四个问题：管什么 / 要什么 / 怎么做 / 交什么。

### 1. 第一段：Skill 的逻辑框架（约 400 字 + 一张表）
- 四格框架表（上面那张）
- 一段补充："这四格其实就是你接到一个项目时脑子里默默回答的四个问题，做 Skill 只是把它落到文字上"
- 一句过渡："下面用两个真实案例验证这个框架是否能套用"

### 2. 第二段：案例 ① 专有名词解释库（约 600 字）

**结构**：业务问题（80 字）→ 套框架（四格表，每格 1–3 行）→ 效果（80 字）

业务问题草稿：营销工作高频用 Excel（产品信息包、WBS、术语表），AI 直接读 `.xlsx` 不稳定，每次都要手动复制粘贴或截图。希望 Excel 一改完，AI 立刻能读到最新版。

四格内容草稿（在 `EXCEL_SYNC_README.md`、`excel-sync.mdc`、`excel_sync_config.json` 三个文件基础上提炼）：
- **边界**：自动把 Excel 同步成 AI 可读的 Markdown，不改原 Excel、不做内容判断
- **输入**：① 待处理：项目内任意 `.xlsx / .xlsm`；② 参考标尺：`excel_sync_config.json`（追踪哪些 / 排除哪些）
- **思考与行动**：执行类资源（`watch_excel.py` 监听 + `excel_sync.py` 单次同步）+ Cursor Rule（`excel-sync.mdc` 告诉 AI 优先读 `cursor_excel/`，别去碰 `.xlsx`，过时就跑 `Excel: sync once`）
- **输出**：`cursor_excel/index.md` 总索引 + 每个 Sheet 一个 `.md`，作为后续 AI 任务的"可读资料库"

效果一句话：以前每次让 AI 看 Excel 要手动转格式，现在打开项目就自动同步，AI 直接读 Markdown 做汇总 / 对比 / 拆解。

### 3. 第三段：案例 ② 说明书重复校对（约 700 字）

**结构同上**：业务问题（120 字，讲清三类错误和硬节点压力）→ 套框架（四格表）→ 效果（80 字）

四格内容草稿（基于 `需求分析_文档校对与信息同步.md` 和 `在线文档校对_Prompt设计.md`）：
- **边界**：北美区文档差异检测器，三条铁律——只读不改 / 只用 Product Info / 只管北美；不评价翻译优美度、不碰法律警示语
- **输入**：① 待处理：校对目标 Excel（说明书 / 包装 / POP）；② 参考标尺：产品信息包（必须）+ 术语表（推荐）+ 变更文档（推荐）
- **思考与行动**：四步 SOP（定位基准 Sheet → 扫描校对范围 → 按三类错误逐行查 → 输出报告）+ 关键判断规则（卖点漏项不算错 / 绿色标注优先 / 按列内容判断而非 Sheet 名字）
- **输出**：差异报告（基准确认 + 校对范围 + 差异清单分三类 + 疑点 + 总结），精确到 Sheet 名 + 行号 + 基准值 + 实际值

效果一句话：从"数小时全量人肉看"变成"20 分钟过差异清单"，PMKT 拿着报告倒逼上游规范变更日志。

### 4. 收尾（约 150 字）
两个案例对比的一行总结：一个是"准备战场"（把素材搬到 AI 能用的地方），一个是"打仗"（直接出差异报告），但都在做同一件事——**让 AI 替人做机械活，人专注做判断**。鼓励同事按这四格回头看自己的工作，找出值得沉淀的 Skill。
