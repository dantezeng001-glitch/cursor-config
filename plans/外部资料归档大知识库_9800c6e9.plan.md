### Step 3 — 跑标准归档流程

按 [命名与归档规范.md](c:\Users\016551\OneDrive\Desktop\科技树\知识库及前端开发\知识库\运行域\治理文档\命名与归档规范.md) 第 §收件箱批量归档验收：

1. `python 运行域/脚本/archive_inbox.py --dry-run` 预演全量
2. 逐文件确认归档目标，写入 `_建议归档.md`
3. 执行归档（高置信自动 + 中低置信人工确认）
4. 再跑 `--dry-run` 验证收件箱只剩 README 和 _建议归档

## 四、风险与边界

- **不动原文件**：4 个外部文件夹保持原位，本次只做"复制 + 归档"
- **重复识别**：归档时发现同一份内容已在知识库存在，原则是新版本 supersede 旧版本，旧版本写 frontmatter `supersedes` 字段
- **WBS 是否真值得保留**：建议归档完后回头扫一眼 `知识域/策略判断/历史规划/` 下已有 WBS，如果 NT、UAS 这些项目其实已经过期或被新版替代，可以一起进 supersedes
