# NT 上市 WBS 归档计划

## 目标

让 `知识库/收件箱/北美NT上市WBS表.xlsx` 一次到位地走完“分类 / 改名 / 同步 / 沉淀为知识域正文”全链路，落到 `知识域/策略判断/历史规划/`。顺手把自动分类规则补齐，避免以后同类文件又要手工 `[e]` 指定目录。

## 决策

- 归类：作为北美一次上市的历史规划档案 → `知识域/策略判断/历史规划/`
- 自动规则：同时补 `NT → OpenFit Pro` 以及 `上市WBS / 上市计划 → 策略判断/历史规划`

## 1. 补齐 archive_inbox.py 的分类规则

文件：[知识库/运行域/脚本/archive_inbox.py](知识库及前端开发/知识库/运行域/脚本/archive_inbox.py)

在现有 `CATEGORY_RULES` 中做两处修改，保持“先匹配历史规划，再匹配产品线代号”的顺序，避免带 WBS 的产品上市档案被错误拉进 `产品主张/OpenFit Pro/`：

- 扩展“历史规划/立项材料”规则，补上 WBS / 上市计划类关键词：
  - 现状：`(r"立项书|规划书|前人材料|建设中", "知识域/策略判断/历史规划", "命中历史规划/立项材料")`
  - 调整后：`(r"立项书|规划书|前人材料|建设中|上市WBS|上市计划|WBS表", "知识域/策略判断/历史规划", "命中历史规划/立项材料")`
- 在其他产品线代号规则旁，追加 `NT → OpenFit Pro` 的自动归类：
  - 新增：`(r"\bNT\b|OpenFit Pro|OpenFitPro|OpenFit-Pro", "知识域/产品主张/OpenFit Pro", "命中产品线：OpenFit Pro")`

这样 `北美NT上市WBS表.xlsx` 会**先命中 WBS 规则**而不是 NT 规则，不会被错判为产品线物料。

## 2. 走一遍 北美NT上市WBS表.xlsx 的全流程

### 2.1 自动分类

- 命中：`立项书|规划书|...|上市WBS|上市计划|WBS表`
- 建议目录：`知识域/策略判断/历史规划`
- 交互：`archive_inbox.py` 会打印建议并等待 `[y]` 确认

### 2.2 文件重命名

由 `build_import_filename` → [kb_naming_rules.py](知识库及前端开发/知识库/运行域/脚本/kb_naming_rules.py) 的 `build_canonical_title` 处理：

- `normalize_product_mentions`：`NT` → `OpenFit Pro`
- `sanitize_file_stem`：在中英文、数字交界处补空格
- 结果文件名：`北美 OpenFit Pro 上市 WBS 表.xlsx`
- 落点：`知识域/策略判断/历史规划/北美 OpenFit Pro 上市 WBS 表.xlsx`

### 2.3 同步为 Markdown

由 [sync_binary_docs.py](知识库及前端开发/知识库/运行域/脚本/sync_binary_docs.py) 执行：

- 运行域副本：每个 sheet 抽取为 `运行域/同步文档/<file_id>/<sheet_name>.md`，只作为追溯缓存
- 知识域沉淀页（由 `write_standard_md_entry` 生成）：
  - 路径：`知识域/策略判断/历史规划/北美 OpenFit Pro 上市 WBS 表_同步版.md`
  - 行为：现在这份 md 会**把运行域各 sheet 的正文直接嵌入**，而不是只留跳转链接

### 2.4 frontmatter 推断

在 `策略判断` 目录下，`audit_document` 会给出：

- `kb_type: decision`
- `related_products: [OpenFit Pro]`（由 `find_related_product` 识别到 NT → OpenFit Pro 的映射）
- `source_docs`：原 xlsx 路径 + 各 sheet 的运行域 md 路径
- `naming_status`：预计为 `review` 或 `compliant`（取决于正文长度与 WBS 表格结构）

### 2.5 前端可见性

- 前端只索引 `知识域/`，所以用户在浏览和搜索里看到的是这份 `_同步版.md`
- 运行域里的 sheet 拆页不进前端主链路，只能通过“来源追溯”区看到
- 评论会挂在 `doc_id` 上，不会因为以后改名丢失

## 3. 执行动作清单

按这个顺序跑：

- 编辑 [archive_inbox.py](知识库及前端开发/知识库/运行域/脚本/archive_inbox.py)：调整“历史规划”规则，新增 OpenFit Pro 规则
- 在知识库根目录执行 `python 运行域/脚本/archive_inbox.py --file 北美NT上市WBS表.xlsx`
- 交互时：
  - 对“命中历史规划/立项材料”建议按 `[y]`
  - 同步问询时按 `[Y]` 立即转 Markdown
  - 源 xlsx 是否删除，由你决定（建议先保留作备查）
- 跑 `python 运行域/脚本/refresh_index.py` 和 `python 运行域/脚本/audit_kb_naming.py` 收尾

## 4. 风险与校验点

- WBS 属于大表，Excel 抽取后的表格可能较宽，需检查 `_同步版.md` 是否因表格过宽在前端展示不友好
- 如果 xlsx 里包含多个 sheet（如分阶段/分职能），同步会把它们都嵌进同一页，需要人工看一下结构清晰度
- 新加的 `NT → OpenFit Pro` 自动规则，会影响未来所有含 NT 的文件名；如果担心误伤，可先只加 WBS/上市计划那条，NT 那条等看一次实际触发情况再加
