## 一、产出物清单

三张 PNG，存到 [`0513 Skills分享/`](0513%20Skills%E5%88%86%E4%BA%AB/) 下，命名走"图N-用途.png"，方便后续维护：

- `图1-Skill逻辑框架.png`
- `图2-术语库案例-BeforeAfter.png`
- `图3-中英校对案例-BeforeAfter.png`

## 二、风格统一约定

- **横向构图**，4:3 比例（适合钉钉文档展示，也兼顾后续 PPT 复用）
- **配色克制**：3-4 种主色，避免彩虹色；类似"editorial / 商务汇报风"，不要 SaaS 渐变也不要 AI slop 玻璃拟物
- **中文标签为主**，FABE、Skill、AI 等术语保留英文
- **不用 emoji**，所有图标用线条 icon
- **不堆装饰**——核心信息能一眼看懂，留白足够

## 三、每张图的具体设计

### 图 1：Skill 逻辑框架（逻辑图）

**核心要表达**：四格框架的并列关系 + "边界统领其他三格"的层级感。

构图：横向四列方框 / 或上方一行"边界"统领下方三格"输入→思考→输出"的流向。

倾向后者，因为更能呼应文档里"任务边界是最容易忽略但最关键的一格"的判断：

```
┌─────────────────────────────────────────┐
│       ① 任务边界（管什么 / 不管什么）         │
└─────────────────────────────────────────┘
        │                │                │
        ▼                ▼                ▼
   ② 输入物料  →   ③ AI 思考行动  →  ④ 输出格式
   (待处理+标尺)    (步骤+判断)       (字段+结构)
```

每格内放 1 个 icon + 标题 + 一行说明（对应文档第一段表格里的内容）。底部一行小字：**"Skill = 把人做事的方式写下来交给 AI"**

### 图 2：案例①术语库（Before / After）

**核心要表达**：从"老员工脑子里隐形词典"到"全员可查的术语库"。

构图：左右分屏，中间用箭头 + "Skill" 标签衔接。

- **左侧 Before**：
  - 一个老员工头像 + 头顶大脑图标里写满黑话（FABE / KSP / MSG House…）
  - 一个校招生头像 + 头顶问号
  - 一个协作部门图标 + 反复追问的对话气泡

- **中间过渡**：箭头 + 标签"术语库提取 Skill"

- **右侧 After**：
  - 一份 Excel 文档图标（带 5 列示意）
  - 标注"约 90 条术语 / 5 大分类"
  - 新人 + 协作部门图标共同指向这份 Excel

### 图 3：案例②中英校对（Before / After）

**核心要表达**：从"数小时双屏肉眼比对"到"20 分钟过差异清单"。

构图：同样左右分屏 + 中间 Skill 衔接。

- **左侧 Before**：
  - 双屏图标（左屏中文版、右屏英文版）
  - 人物图标在两屏间来回看（眼神疲惫）
  - 标注"18+ Sheet / 3-4 轮 / 数小时"

- **中间过渡**：箭头 + 标签"中英校对 Skill"

- **右侧 After**：
  - 中文版 + 英文版两个文档图标 → AI 图标 → 差异报告图标
  - 差异报告示意三类错误分表
  - 标注"20 分钟出清单 / 人只做判断"

## 四、插入 md 的位置

每个二级标题下方第一行就插图，形成一致的视觉节奏：

- `## 一、Skill 的逻辑框架：四格` 标题下 → 图1
- `## 二、案例 ①：专有名词解释库` 标题下 → 图2
- `## 三、案例 ②：说明书重复校对` 标题下 → 图3

md 引用语法：

```markdown
![Skill 逻辑框架](图1-Skill逻辑框架.png)
```

钉钉粘贴 md 时如果不自动加载图片，需要你手动把这三个 PNG 拖进文档对应位置——这一步无法绕过，因为钉钉文档不读本地路径。

## 五、生成方式与质量预期

- 用 `GenerateImage` 工具生成，每张图独立 prompt
- 一次性给三张图打底，看完整体效果后再决定要不要返工某一张
- 如果对色调 / 风格不满意，告诉我"太花""太单调""换成 XX 风格"之类的方向，我重生成对应图即可，不用全部重做
