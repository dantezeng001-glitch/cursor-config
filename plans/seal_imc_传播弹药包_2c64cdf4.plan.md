# SEAL Swimming IMC 传播弹药包 — 实施计划（V2）

## 一、角色与边界

### PMKT 只做"技术—营销翻译层"

PMKT 的不可替代价值：把技术原理翻译成"各 POC 可以直接取用的原材料"。
PMKT 不做成品文案生产——KOL Brief、PR Pitch、Social Caption、EDM、UGC 玩法是各 POC 的专业领域，强行替写会降低成品质量。

### PMKT 做 / 不做 清单


| PMKT 做                   | PMKT 不做（属于 POC）                           |
| ------------------------ | ----------------------------------------- |
| 技术事实梳理（痛点—机制—结果）         | KOL Brief 成品（Sonia / Eva）                 |
| 分层口径（硬核 / 场景 / 情绪三档示范段）  | PR Pitch Email 成品（Christina）              |
| 核心比喻库                    | EDM Subject + 正文成品（Gemma）                 |
| 可视化机会提示（机制怎么被看见）         | Social Caption / 短视频文案成品（Yanky）           |
| Claim 证据锚点（规格 / 数据 / 文献） | UGC Challenge 玩法 SOP（Thea + Content）      |
| 红线清单（不能说什么、改说什么）         | 官网 / Pillar Page 全文（Content-Eugene + Web） |
| 跨 CM 口径一致性               | 投放结构（DM-Winnie）、视觉 brief（Cara）            |

## 三、每个 Core Message 子包的四大模块

### 模块 1 — 技术事实一页纸（Fact Sheet）

- **痛点—机制—结果** 三段式严谨叙述（300–500 字）
- **关键规格数字**（与 [Spec Sheet](C:\Users\016551\OneDrive\Desktop\SwimPro\04_子项目_0607IMC活动English]OpenSwim Pro Spec Sheet-20260119.md) 对齐）
- **可引用锚点**：数据、实验室结论、学术文献、标准
- **与上一代 / 竞品的技术差分点**（若有）

目的：给所有 POC 一份"技术真相的单一事实来源"，任何人写的成品都可以回溯到这里对账。

### 模块 2 — 分层口径（Message Ladder）

三档各给"**一段示范段落 + 一组关键英文词汇**"：


| 档       | 给谁用                        | 形态                        |
| ------- | -------------------------- | ------------------------- |
| **硬核档** | PR 垂媒、硬核测评 KOL、合规审核        | 保留"机械阻抗""堵耳效应""DSP EQ"等术语 |
| **场景档** | 安利型 KOL、EDM、官网 Pillar Page | 用户痛点引入 + 效果描述，少术语         |
| **情绪档** | 社媒短视频、UGC Hook、广告 Hook     | 比喻 + 感官动词，无术语             |


目的：各 POC 按自己渠道的调性直接"按档位取材"，不需要自己翻译一遍技术。

### 模块 3 — 比喻库与可视化机会

- **核心比喻**：2–4 组可直接被创作者借用的比喻（CM#1 已有 5 组："推车与陷泥""拍打与抚摸""开窗与闷罐""抢戏与聚光灯""浓雾与探照灯"；其余 3 个 CM 需补齐）
- **可视化机会**：技术机制如何"被看见" 的提示，例：
  - CM#1 堵耳效应 → "戴耳塞 vs 不戴耳塞水下对比测听" 的镜头逻辑
  - CM#3 30° 倾斜 → "普通耳机 vs SEAL 的面部压强触感对比" 的体验逻辑

不给成品 brief，只给"机制可视化的切入口"，由 Content / Cara / KOL 团队拿去做画面。

### 模块 4 — 红线清单（Do / Don't）

- **"不能说 X，因为 Y，改成 Z"** 三列表
  - 例：CM#1 不要直接说 "water amplifies the sound" → 因为物理真相是"堵耳效应放大低频反而产生闷罐感"被 EQ 反向削掉 → 可改成 "engineered to tame what water does to sound"
- **易被 KOL / 媒体追问的问题 + 标准回答**
  - 例："SEAL 是物理放大音质还是算法补偿？" 标准回答长短两版

目的：避免 copy 与技术真相打架被硬核测评 KOL 或合规质疑。
