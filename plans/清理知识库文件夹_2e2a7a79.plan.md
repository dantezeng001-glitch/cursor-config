# 统一表述知识库文件夹清理方案

## 当前问题

**1. 同一产品的资料散落在 3 个文件夹里**

- 想找 OpenDots ONE 的全部资料，需要分别去 `Meet&Spec md文件/OpenDots ONE/`、`POP md文件/` 里翻
- 没法一眼看到"这个产品有什么、缺什么"

**2. 文件命名风格混乱**（同一类文件，命名方式五花八门）

- Meet Sheet: `Shokz Openrun meet.md` / `[English]OPENSWIM PRO_MEET.md` / `US_OpenMeet_Meet.md` / `MEET已校对0725.md`
- Spec Sheet: `Shokz Openrun Spec.md` / `OPENFIT Spec Sheet 锁定-仅能修改价格.md` / `OpenRun Mini_Spec_Sheet.md`
- 产品名大小写不统一: `Opencomm UC/` vs `OpenComm 2UC/` vs `OpenComm/`

**3. 有一些冗余/过时内容**

- `wiki/` 文件夹（15 个文件）— 你已确认不是必需的
- `fix_no_source.py` — 临时修复脚本，已完成使命
- `__pycache__/` — Python 缓存
- `OpenComm2 2025 Upgrade/` — 只有一个文件，和 `OpenComm 2UC/` 内容重叠

**4. 根目录杂乱**

- Python 脚本、文档、Excel 输出混在一起，打开文件夹第一眼看不出重点

## 清理方案

### 整理后的文件夹结构

```
统一表述知识库/
├── 项目指导文档.md              <- 项目说明（保留在最显眼位置）
├── 产品资料索引.md              <- 产品全景图
├── Shokz统一表述知识库.xlsx     <- 最终输出物
│
├── 新文件收件箱/                <- 拖入新文件的入口
│
├── 产品源文件/                  <- [核心改动] 所有产品资料按产品归类
│   ├── OpenDots ONE/
│   │   ├── [EN] Meet.md
│   │   ├── [FR] Meet.md
│   │   ├── [EN] Spec.md
│   │   ├── [FR] Spec.md
│   │   ├── POP.md
│   │   └── 原始文件/           <- 如有 PDF 等原始文件
│   ├── OpenFit/
│   │   ├── [EN] Meet.md
│   │   ├── [EN] Spec.md
│   │   └── POP.md
│   ├── OpenFit Pro/
│   ├── OpenRun/
│   ├── OpenRun Pro 2/
│   ├── ...
│   └── 通用/                   <- 非特定产品的 POP（CCT、BOS、水箱等）
│       ├── POP_CCT.md
│       ├── POP_BOS.md
│       └── ...
│
└── 工具/                       <- Python 脚本集中存放
    ├── generate_excel.py
    ├── check_consistency.py
    └── auto_extract.py
```

### 具体清理动作

**动作 1：创建新结构并迁移文件**

- 新建 `产品源文件/` 目录，按产品名建子文件夹
- 把 `Meet&Spec md文件/` 和 `POP md文件/` 中的文件迁移到对应产品文件夹下
- `Meet&Spec 文件/` 中的 PDF 放到对应产品的 `原始文件/` 子文件夹
- 文件统一重命名为：`[EN] Meet.md`、`[FR] Spec.md`、`POP.md` 等简洁格式

**动作 2：处理特殊情况**

- OpenComm 2UC 有两个文件夹（`OpenComm 2UC/` 和 `OpenComm2 2025 Upgrade/`），合并到 `OpenComm 2 UC/`
- Opencomm UC 有多个版本（100/110），保留为 `[EN] Meet (100).md`、`[EN] Meet (110).md`
- 非特定产品的 POP（CCT、BOS、BOS+、SEAL、水箱、助听器、其他类型文案）放到 `通用/`

**动作 3：删除冗余**

- 删除 `wiki/` 整个文件夹（15 个文件）
- 删除 `fix_no_source.py`
- 删除 `__pycache__/`（如存在）

**动作 4：脚本集中管理**

- 把 `generate_excel.py`、`check_consistency.py`、`auto_extract.py` 移到 `工具/` 文件夹
- 更新脚本中的路径引用（`Meet&Spec md文件` -> `产品源文件`，`POP md文件` -> `产品源文件`）

**动作 5：更新配套文件**

- 更新 `项目指导文档.md` 中的文件结构说明
- 更新 `产品资料索引.md`
- 更新 Cursor 规则中的路径引用（`inbox-process.mdc`、`knowledge-base-nav.mdc`）

### 风险控制

- 所有操作先在文件系统中创建新结构并复制文件，确认无误后再删除旧文件夹
- 脚本路径更新后先测试 `generate_excel.py` 能否正常生成 Excel
- 旧文件夹在确认一切正常前不删除
