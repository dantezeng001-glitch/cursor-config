# Layout Grid · 1280×720 网格

> 所有 Shokz deck 的版式锁。**关键点位的 `top` 值不能擅自改**——这是品牌排版的"信用印章"，改了一处全 deck 就和原品牌脱钩。

---

## 1. 画布

| 维度 | 值 |
|---|---|
| 宽 | `1280px` |
| 高 | `720px` |
| 比例 | 16:9（与 PowerPoint 标准画布一致） |
| 背景 | `#FFFFFF`（默认），或 `#050505` Ink 整页（章节分隔 / 致谢 / 强调页） |

### 1.1 适配窄屏

`.deck-slide` 在视口 < 1360px 时自动 scale，保持比例。这套规则已经写在 [`assets/deck.css`](../assets/deck.css) 里，不需要在每页重写。

---

## 2. 安全边距

| 区域 | 位置 |
|---|---|
| 左 padding | `96px`（所有内容左边界） |
| 右 padding | `96px`（所有内容右边界） |
| 内容可用宽 | `1088px`（= 1280 - 96 - 96） |
| 顶 padding | `32px`（页眉位置） |
| 底 padding | `46px`（页脚距底 = 720 - 674） |

> 这意味着所有"业务内容"都在 `(96, 32) → (1184, 696)` 这个矩形内。

## 4. 横向布局规则

### 4.1 单栏

- 整宽 `1088px`，从 `left: 96px` 起
- 适用：标题区、全宽金句条、整页表格

### 4.2 二栏（533/533）

- 栏宽 `533.33px`，间距 `21.33px`
- 左栏起点 `left: 96px`，右栏起点 `left: 650.66px`
- 适用：STAR + Result 对比 / 基础项目侧 vs AI 杠杆侧

### 4.3 三栏（350/350/350）

- 栏宽 `350.22px`（精确值 `350.22 / 350.23 / 350.23`），间距 `18.66px`
- 左 `left: 96px` / 中 `left: 464.87px` / 右 `left: 833.77px`
- 适用：STA 三件套 / GOAL-ACTION-METRIC / 三层涟漪 / 心路历程 3 列

### 4.4 四栏（208/257/208/208）+ 间距

- 来自原 deck 时间线 P02
- 列 1：`left: 96, width: 208.89`
- 列 2：`left: 364.08, width: 257.34`
- 列 3：`left: 677.15, width: 208.89`
- 列 4：`left: 995.29, width: 208.89`
- 适用：4 段时间线 era（教育-工作-当前）

### 4.5 锚点 + 网格混合（240 + 3×262）

- 来自原 deck P05 九件事页
- 左侧 3 大类锚点：`left: 96, width: 240, height: 140`，垂直堆 3 个
- 右侧 3×3 数据卡：每张 `width: 262, height: 140`，起点 `left: 356, 638, 920`
- 适用：多任务/多项目全景页

### 4.6 二栏（320/748）

- 来自原 deck P08 案例页
- 左：`left: 96, width: 320`（S/T 卡叠放）
- 右：`left: 436, width: 748`（A 卡 + 图片）
- 适用：上下文重的左侧 + 大块 action 的右侧

## 5. 网格违规检测

写完一页后用以下规则自检：

| 现象 | 可能违规 |
|---|---|
| 内容左边界 ≠ 96 | 没走标准左 padding |
| 内容右边界 > 1184 | 超出右 padding |
| 页眉 top ≠ 32 / 页脚 top ≠ 674 | 没走标准 chrome |
| 主标题 top ≠ 85（±2px 容差）| 没走标准 H1 位置 |
| 副标题 top ≠ 165（±2px）| 没走标准副标位置 |
| 出现 `border-radius: NNpx` 且 N > 0 | Shokz 是直角，违规 |
| 出现栏宽不是 533 / 350 / 262 / 208 这几档 | 不在网格内 |
| 三栏布局栏间距 ≠ 18-21px | 不在网格间距体系内 |

## 6. 密度区间

> 单页密度由"信息单元数 / 内容区可用高"决定。Shokz 风格偏中高密度（管理层汇报场景）。

| 密度 | 卡片/单元数 | 单页字数 | 何时用 |
|---|---|---|---|
| Low | 1-2 个大块 | < 80 字 | 封面 / 章节分隔 / 致谢 / 关键金句页 |
| Mid-Low | 3-4 个单元 | 80-200 字 | 个人简介 / 概念页 / 收尾页 |
| Mid | 5-8 个单元 | 200-400 字 | 标准案例页（STAR 单页）/ 三栏对比 / GOAL-ACTION-METRIC |
| Mid-High | 9-12 个单元 | 400-700 字 | 9-cards Overview / 时间线+心路 / 三层涟漪 + 复盘条 |
| High | 13+ 个单元 | 700+ 字 | 极少用——优先拆成两页 |

### 6.1 反密度异常

- ❌ 整页只有一个标题 + 一行字（除非是有意的章节页）
- ❌ 单页超过 12 个独立卡片（信息过载，目检会眼花）
- ❌ 同一页混用 5 种以上颜色（Era 色已用 3 种就别再加灰阶层级）
- ❌ 连续 3 页都是 Mid-High 密度（视觉疲劳；要插入 Low 密度的金句页换气）

## 7. Z 轴（不常用但要知道）

- `.deck-toolbar`（右上角导航）：`z-index: 10`
- 默认 deck 元素：无 z-index，按 DOM 顺序叠
- 浮层（如 hover tip）：不要在 deck 里用——deck 是静态文档，没有 hover 交互
