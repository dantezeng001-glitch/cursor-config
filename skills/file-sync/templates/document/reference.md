# Reference

## Required Project Files

- `tools/document_sync.py`: scans PDF, PPT, and PPTX files and exports each into `cursor_docs/` as a single `document.md` (plus optional `cleaned.md` for OCR PDFs), and an `index.md`
- `tools/watch_documents.py`: polls for document or config changes and reruns sync
- `tools/convert_legacy_ppt.ps1`: converts `.ppt` to `.pptx` on Windows when PowerPoint is installed
- `document_sync_config.json`: source patterns, exclude patterns, OCR settings
- `.cursor/rules/document-sync.mdc`: tells Cursor to prefer synced document artifacts over direct binary reads
- `.vscode/tasks.json`: includes `Docs: sync once` and `Docs: watch sync`
- `DOCUMENT_SYNC_README.md`: explains usage inside the project

## Default Dependencies

### Required

- `pypdf`
- `python-pptx`

### Recommended

- `pymupdf` — significantly better CJK text extraction from PDFs (used as primary extractor when available; falls back to `pypdf` if missing)

### Optional

- `pytesseract` + `Pillow` — OCR fallback for scanned PDF pages
- Tesseract OCR binary in `PATH`

## OCR Notes

- OCR is used only when a PDF page has no extracted text.
- OCR quality depends on a local Tesseract installation and language packs.
- Default OCR languages are `chi_sim+eng`.

## Legacy PPT Notes

- `.pptx` is fully supported directly.
- `.ppt` requires Windows PowerPoint automation.
- If PowerPoint is unavailable, the installer should still finish, but explain that legacy `.ppt` files may need manual conversion to `.pptx`.

## Merge Guidance

### Existing `.vscode/tasks.json`

- Preserve unrelated tasks.
- Add or update these task labels:
  - `Docs: sync once`
  - `Docs: watch sync`

### Existing `.cursor/rules/`

- Add `document-sync.mdc`.
- Do not remove unrelated rules.

### Existing `tools/`

- Reuse the folder.
- Overwrite only the document sync scripts if the user asked to install or refresh the workflow.

## Post-Extraction Cleanup

After sync produces `.md` files, the md-cleanup skill's script is invoked automatically to fix encoding artifacts (CJK radical characters, NFKC normalization, rich-text export residuals, etc.):

```bash
python "~/.cursor/skills/md-cleanup/scripts/md_cleanup.py" "<output.md>" --in-place
```

This is especially important for:

- Chinese-language PDFs (Kangxi radical substitution is common)
- PDFs generated by design tools (Figma, Canva) that use Type3 fonts
- Any PDF where extracted text contains visually-similar-but-wrong CJK characters

The sync template (`template_document_sync.py`) applies NFKC normalization during extraction and then calls md-cleanup as a post-processing step (best-effort; silently skipped if md-cleanup skill is not installed). The md-cleanup skill covers character-level residuals plus rich-text export artifacts that NFKC does not reach.
