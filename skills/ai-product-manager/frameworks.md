# 分析框架库

## 一、需求发现框架

### 常见业务问题分类

帮助快速定位用户问题属于哪个类型，从而选择追问方向：

| 问题类型 | 典型表述 | 追问方向 |
|---------|---------|---------|
| 效率问题 | "太慢了""来回太多次" | 还原流程步骤 → 找瓶颈环节 → 量化耗时 |
| 质量问题 | "总出错""不一致""感觉不对" | 要具体案例 → 对标参照 → 找质量断点 |
| 管理问题 | "没人管""乱""经常漏" | 理清职责 → 检查信息流 → 找断裂点 |
| 协作问题 | "沟通成本高""信息不同步" | 画出协作链路 → 找信息断裂点 → 量化等待时间 |
| 能力问题 | "不知道怎么做""每个人做法不一样" | 检查有无 SOP → 培训机制 → 标准化程度 |
| 工具问题 | "工具不好用""功能缺失" | 先确认是工具问题还是流程问题 → 避免用工具补流程的坑 |

### 问题深度检验

如果追问后发现问题仍然是"表面级"的，用这几个测试深挖：

- **5 Why 测试**: 对根因连续追问 5 次"为什么"，直到无法再拆解
- **反事实测试**: "如果这个问题不存在了，你的工作会有什么不同？" — 如果用户说不出具体差异，说明问题定义还不够清晰
- **利益相关方测试**: "除了你自己，还有谁也觉得这是个问题？" — 如果只有一个人觉得是问题，可能不是最高优先级

## 二、流程分析框架

### 流程成熟度评估

| 等级 | 名称 | 特征 |
|------|------|------|
| L1 | 初始级 | 靠人脑记忆，无文档，结果因人而异 |
| L2 | 已定义 | 有 SOP，但执行不一致 |
| L3 | 已管理 | SOP 稳定执行，有质量检查点 |
| L4 | 已量化 | 有数据度量，能识别瓶颈 |
| L5 | 持续优化 | 基于数据主动优化，闭环改进 |

### ECRS 分析法

| 原则 | 含义 | 示例问题 |
|------|------|---------|
| **E**liminate | 能不能不做？ | 这个审批环节是否必要？ |
| **C**ombine | 能不能合到一起做？ | 两次人工检查能否合为一次？ |
| **R**earrange | 能不能换个顺序？ | 先翻译再审核 vs 先审核再翻译？ |
| **S**implify | 能不能更简单？ | 能否用模板减少人工填写？ |

### 价值流图关键指标

| 指标 | 定义 | 用途 |
|------|------|------|
| 周期时间 | 单个环节的处理时间 | 找最慢环节 |
| 等待时间 | 环节之间的排队时间 | 找隐性浪费 |
| 前置时间 | 端到端时间 | 整体效率 |
| 一次通过率 | 一次做对的比例 | 质量水平 |

## 三、优先级排序框架

### RICE 评分法

| 功能/项目 | Reach | Impact (1-3) | Confidence (%) | Effort (人天) | RICE 分 |
|-----------|-------|-------------|----------------|-------------|---------|

RICE = (Reach × Impact × Confidence%) / Effort

**Impact**: 3=极大 / 2=较大 / 1=一般 / 0.5=微小
**Confidence**: 100%=有数据 / 80%=有反馈 / 50%=直觉

### MoSCoW 分类法

| 分类 | 含义 |
|------|------|
| **Must** | 没有就无法运转 |
| **Should** | 重要但可延后 |
| **Could** | 锦上添花 |
| **Won't** | 本期不做 |

### 紧急-重要矩阵

|  | 紧急 | 不紧急 |
|--|------|--------|
| **重要** | 立即做 | 规划做 |
| **不重要** | 委派/简化 | 不做 |

## 五、变更管理框架

### ADKAR 模型

| 阶段 | 含义 | 关键动作 |
|------|------|---------|
| **A**wareness | 认知：为什么要变 | 沟通背景和必要性 |
| **D**esire | 意愿：愿意参与 | 说明对个人的好处 |
| **K**nowledge | 知识：知道怎么做 | 培训、SOP、FAQ |
| **A**bility | 能力：做得到 | 试点、辅导、工具支持 |
| **R**einforcement | 巩固：持续做 | 考核、反馈、持续优化 |

## 六、效率度量框架

### 流程效率指标

| 指标 | 定义 | 计算公式 |
|------|------|---------|
| 任务吞吐量 | 单位时间完成数 | 完成数 / 时间段 |
| 平均处理时间 | 单任务平均耗时 | 总耗时 / 任务数 |
| 一次通过率 | 无需返工的比例 | 一次通过数 / 总提交数 |
| 自动化率 | 自动化环节占比 | 自动化步骤数 / 总步骤数 |

### 内容产出效率指标

| 指标 | 定义 |
|------|------|
| 内容生产周期 | Brief → 终稿的天数 |
| 翻译周转时间 | 原文定稿 → 译文交付的天数 |
| 评审通过率 | 首次评审即通过的比例 |
| 术语一致率 | 内容中术语与术语库一致的比例 |
| 内容复用率 | 复用已有素材的比例 |
