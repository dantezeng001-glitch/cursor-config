# 产品经理文档模板库

## 一、需求发现阶段

### 问题摘要模板

```markdown
## 问题摘要（请确认）

**核心问题**: [一句话]
**问题背景**: [2-3 句话]
**受影响方**: [谁在承受这个问题]
**当前状态**: [现在怎么做的]
**关键痛点**:
1. [痛点 1]
2. [痛点 2]
**期望目标**: [做成什么样算成功]
**约束条件**: [不能动的 / 资源限制]
**建议方向**: [建议从 XX 模块入手]

> 以上理解是否准确？有需要补充或纠正的吗？
```

### 需求发现检查清单

追问结束前，确认以下信息是否已收集完整：

- [ ] 有至少一个具体的问题场景/案例
- [ ] 清楚当前流程的关键步骤
- [ ] 识别了瓶颈/痛点环节
- [ ] 探索了根因（不只是表面现象）
- [ ] 了解了之前的尝试和失败原因
- [ ] 明确了成功标准
- [ ] 确认了约束和资源

- **品牌调性**: [关键词]

|---------|------|------|------|---------|--------|

### RACI 矩阵

```markdown
| 任务/环节 | 产品 | 市场 | 翻译 | 设计 | 开发 |
|-----------|------|------|------|------|------|
| | R/A/C/I | | | | |

R = Responsible（执行）  A = Accountable（拍板）
C = Consulted（咨询）    I = Informed（知会）
```
